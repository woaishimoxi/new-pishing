# 面向中小型企业的轻量化钓鱼邮件检测与溯源系统 - 完整项目文档

## 一、项目概述

### 1.1 项目名称
面向中小型企业的轻量化钓鱼邮件检测与溯源系统

### 1.2 项目目标
- 提供轻量化、高效的钓鱼邮件检测能力
- 实现邮件攻击溯源追踪
- 支持多维度融合检测（轻量模型 + 规则引擎 + AI分析 + 威胁情报）
- 提供直观的数据可视化大屏

### 1.3 技术栈
| 层级 | 技术 |
|------|------|
| 后端框架 | Python 3.8+, Flask 2.x |
| 机器学习 | LightGBM, IsolationForest |
| 数据库 | SQLite |
| 威胁情报 | 微步在线API |
| AI模型 | 阿里通义千问API |
| 前端 | HTML5, CSS3, JavaScript, Bootstrap 5, Chart.js |

## 三、核心检测流程

### 3.1 数据流图

```
邮件输入 (手动上传/邮箱拉取/手动输入)
    │
    ▼
┌─────────────────┐
│  1. 邮件解析     │ → email_parser.py
│  email_parser   │   提取: 发件人、收件人、主题、正文、URL、附件
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 2. 附件沙箱分析  │ → threatbook.py
│   sandbox       │   微步在线文件沙箱API
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  3. URL分析      │ → url_analyzer.py
│  url_analyzer   │   白名单检查、品牌仿冒检测、可疑参数检测
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  4. 特征提取     │ → feature_extractor.py
│  features       │   39维传统特征 + 35维语义特征 + 26维统计特征
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 5. AI语义分析    │ → alerts.py
│   ai_analysis   │   大语言模型分析邮件内容
└────────┬────────┘
         │
         ▼
┌─────────────────────────────────────────────────────────────┐
│  6. 多模型融合检测 → detector.py                            │
│  ┌─────────────┬─────────────┬─────────────┬─────────────┐ │
│  │ RF分类器    │ XGB分类器   │ 异常检测器  │  规则引擎   │ │
│  │  权重: 1.5  │  权重: 1.5  │  权重: 1.0  │  权重: 1.0  │ │
│  └─────────────┴─────────────┴─────────────┴─────────────┘ │
│  ┌─────────────┬─────────────┐                             │
│  │  AI分析     │ URL分析     │   最终得分 = Σ(分数×权重)/Σ(权重) │
│  │  权重: 1.2  │  权重: 1.0  │                             │
│  └─────────────┴─────────────┘                             │
└─────────────────────────────────────────────────────────────┘
         │
         ▼
┌─────────────────┐
│  7. 阈值判断     │
│                 │   >= 0.60 → PHISHING（钓鱼）
│                 │   >= 0.35 → SUSPICIOUS（可疑）
│                 │   < 0.35  → SAFE（安全）
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  8. 溯源分析     │ → traceback.py
│  traceback      │   IP地理位置、DNSBL黑名单、WHOIS、攻击链
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  9. 结果存储     │ → database.py
│  database       │   SQLite存储告警记录
└─────────────────┘
```

### 3.2 检测模型说明

| 模型 | 类型 | 特征维度 | 权重 | 说明 |
|------|------|----------|------|------|
| RF分类器 | 有监督 | 35维 | 1.5 | 随机森林，基于语义特征 |
| XGB分类器 | 有监督 | 35维 | 1.5 | 梯度提升，基于语义特征 |
| 异常检测器 | 无监督 | 26维 | 1.0 | IsolationForest，检测异常模式 |
| 规则引擎 | 规则 | - | 1.0 | 关键词/模式匹配 |
| AI分析 | LLM | - | 1.2 | 大语言模型语义分析 |
| URL分析 | 规则 | - | 1.0 | URL风险特征分析 |

### 3.3 Kill Switch硬规则

以下情况直接判定为PHISHING（一票否决）：
1. 沙箱检测到恶意代码
2. 包含可执行文件附件
3. **源IP在黑名单中**（已修复逻辑缺陷）
4. 邮件认证全部失败 + 发件人冒充
5. 多个高风险URL
6. 双重扩展名附件
7. 隐藏链接 + 表单组合

## 六、核心算法

### 6.1 特征提取

**35维语义特征（用于RF/XGB）**：
- 主题特征（6维）：紧急程度、威胁性、诱惑性、情感分析
- 发件人特征（2维）：冒充类型、邮箱异常
- 正文特征（16维）：关键词计数、个人信息请求、钓鱼CTA
- URL特征（11维）：域名长度、IP地址、可疑参数

**26维统计特征（用于异常检测器）**：
- 基础统计（6维）：字符数、行数、URL数、主题长度
- URL特征（4维）：高风险URL、链接不匹配、唯一域名
- HTML特征（3维）：HTML、脚本/表单、附件提示
- 钓鱼模式（8维）：伪造发件人、base64、中文关键词
- 邮件认证（5维）：SPF、DKIM、DMARC

### 6.2 融合评分公式

```
最终得分 = (RF分数 × 1.5 + XGB分数 × 1.5 + 异常检测分数 × 1.0 
         + 规则分数 × 1.0 + AI分数 × 1.2 + URL分数 × 1.0) 
         / (1.5 + 1.5 + 1.0 + 1.0 + 1.2 + 1.0)
```

## 七、安装与运行

### 7.1 环境要求

- Python 3.8+
- 依赖包见 requirements.txt
