# 钓鱼邮件检测与溯源系统 - 完整项目文档

## 一、项目概述

### 1.1 项目名称
面向中小型企业的轻量化钓鱼邮件检测与溯源系统

### 1.2 项目目标
- 提供轻量化、高效的钓鱼邮件检测能力
- 实现邮件攻击溯源追踪
- 支持对抗样本测试与鲁棒性评估
- 提供直观的数据可视化大屏

### 1.3 技术栈
- **后端**: Python 3.8+, Flask 2.x, LightGBM
- **前端**: HTML5, CSS3, JavaScript, Bootstrap 5, Chart.js
- **数据库**: SQLite
- **机器学习**: LightGBM (轻量级梯度提升框架)
- **外部API**: VirusTotal, IP-API

#### 3.2.3 Kill Switch规则
```python
def _check_kill_switch(self, features, url_risk_level, url_risk_score):
    # 1. 沙箱检测到恶意代码
    if features.get('sandbox_detected'):
        return "Sandbox Detected Malware"
    
    # 2. 三重认证失败 + 身份冒充
    if spf_fail and dkim_fail and dmarc_fail and display_name_mismatch:
        return "Authentication Failed + Impersonation"
    
    # 3. VirusTotal高检测率
    if features.get('max_vt_detection_ratio', 0) > 0.5:
        return "Known Malicious URL"
    
    # 4. 高风险URL
    if url_risk_level == 'HIGH' and url_risk_score >= 80:
        return f"High Risk URL Detected"
```

#### 3.2.4 规则评分引擎
```python
def _demo_predict(self, feature_vector, email_data):
    score = 0.0
    
    # 规则1: 域名信誉 (权重: 0.25)
    if domain_age < 30: score += 0.15
    
    # 规则2: 邮件认证 (权重: 0.20)
    if spf_fail and dkim_fail and dmarc_fail: score += 0.25
    
    # 规则3: 发件人异常 (权重: 0.15)
    if suspicious_from_domain: score += 0.15
    
    # 规则4: URL风险 (权重: 0.20)
    if ip_url_count > 0: score += 0.15
    
    # 规则5: 文本风险 (权重: 0.10)
    if urgent_keywords > 3: score += 0.10
    
    # 规则6: 附件风险 (权重: 0.10)
    if has_executable_attachment: score += 0.20
    
    # 规则7: HTML风险 (权重: 0.05)
    if has_hidden_links: score += 0.15
    
    # 规则8: 验证码邮件保护
    if is_verification_email: score -= 0.35
    
    confidence = sigmoid(score)
    return is_phish, confidence
```

#### 3.4.2 附件风险检测
```python
HIGH_RISK_EXTENSIONS = {
    '.exe', '.bat', '.cmd', '.com', '.scr', '.vbs', '.js', '.ps1', '.msi', '.hta'
}

MEDIUM_RISK_EXTENSIONS = {
    '.doc', '.docm', '.xls', '.xlsm', '.zip', '.rar', '.7z'
}

def _check_suspicious_file_type(self, filename, content_type):
    ext = '.' + filename.rsplit('.', 1)[-1].lower()
    
    # 高危扩展名
    if ext in HIGH_RISK_EXTENSIONS:
        return True
    
    # 双重扩展名 (如: document.pdf.exe)
    parts = filename.split('.')
    if len(parts) > 2 and parts[-2] in ['.pdf', '.doc', '.xls']:
        return True
    
    return False
```

#### 3.5.2 DNSBL黑名单检测
```python
DNSBL_SERVERS = [
    ('zen.spamhaus.org', 'Spamhaus'),
    ('b.barracudacentral.org', 'Barracuda'),
    ('bl.spamcop.net', 'SpamCop'),
    ('dnsbl.sorbs.net', 'SORBS'),

def _check_blacklist_parallel(self, ip):
    # 反向IP查询
    reversed_ip = '.'.join(ip.split('.')[::-1])
    
    # 并行查询多个DNSBL服务器
    with ThreadPoolExecutor() as executor:
        futures = {
            executor.submit(self._query_single_dnsbl, reversed_ip, server, name)
            for server, name in DNSBL_SERVERS
        }
        
        for future in as_completed(futures):
            is_listed, server_name = future.result()
            if is_listed:
                result['blacklists'].append(server_name)
    
    return result
```

```python
def analyze_single_url(self, url, check_whitelist=True, use_vt=True):
    result = {
        'url': url,
        'risk_level': 'UNKNOWN',
        'risk_score': 0,
        'is_whitelisted': False,
        'domain_age_days': None,
        'suspicious_keywords': [],
        'brand_abuse': [],
        'virustotal': None
    }
    
    risk_score = 0
    
    # 1. 白名单检查
    if self.is_trusted_domain(domain):
        result['is_whitelisted'] = True
        result['risk_level'] = 'SAFE'
        return result
    
    # 2. IP地址URL
    if is_ip_address: risk_score += 40
    
    # 3. 域名年龄
    if domain_age < 30: risk_score += 35
    
    # 4. 可疑关键词
    if suspicious_keywords: risk_score += 50
    
    # 5. 品牌滥用
    if brand_abuse: risk_score += 45
    
    # 6. VirusTotal检测
    if use_vt:
        vt_result = self.check_virustotal(url)
        if vt_result['positives'] > 0:
            risk_score += 50
    
    # 7. 确定风险等级
    if risk_score >= 60: result['risk_level'] = 'HIGH'
    elif risk_score >= 30: result['risk_level'] = 'MEDIUM'
    else: result['risk_level'] = 'LOW'
    
    return result
```

## 十、总结

本系统采用"轻量化模型 + 规则引擎"的混合检测架构，实现了以下核心功能：

1. **多维度特征提取**: 39维特征覆盖邮件头、URL、文本、附件、HTML
2. **三维度融合检测**: 模型预测 + URL风险 + 规则评分
3. **智能降权机制**: 白名单URL、邮件认证通过自动降权
4. **完整溯源追踪**: IP地理位置、黑名单检测、攻击链构建
5. **对抗样本测试**: 支持多种攻击类型模拟和鲁棒性评估

系统专为中小企业设计，具有轻量化、易部署、高精度的特点。
