## 第一章 绪论

### 1.1 研究背景与意义
- 1.1.1 中小企业面临的网络安全现状
  - 钓鱼攻击占网络攻击比例超过90%（引用APWG、Verizon DBIR数据）
  - 中小企业因资源有限成为攻击主要目标
- 1.1.2 传统安全设备在中小企业落地的痛点
  - 硬件成本高：传统邮件网关设备动辄数万元
  - 运维门槛高：需要专业安全团队
  - 资源消耗大：不适合中小企业轻量级IT环境
- 1.1.3 研究轻量化检测系统的必要性
  - 低成本部署需求
  - 高检测精度要求
  - 检测与溯源一体化需求

### 1.2 国内外研究现状
- 1.2.1 邮件检测技术综述
  - 规则匹配方法：SpamAssassin、Rspamd
  - 机器学习方法：朴素贝叶斯、SVM、随机森林
  - 深度学习方法：TextCNN、BERT、GPT
- 1.2.2 邮件溯源技术发展现状
  - 邮件头分析技术
  - IP地理位置追踪
  - DNS黑名单检测（DNSBL）
- 1.2.3 现有研究的不足
  - 重检测轻溯源：多数系统缺少完整溯源能力
  - 模型过于庞大：BERT等模型不适合中小企业部署
  - 缺乏人机协同：无法处理边界样本

### 1.3 论文主要工作与创新点
- **创新点1**：提出了"LightGBM模型 + 规则引擎 + URL分析"的三维度融合检测架构
  - 模型轻量化：参数量小，推理速度快
  - 多维度融合：模型预测(60%) + URL风险(40%)
  - 智能降权：白名单URL、邮件认证通过自动降权
- **创新点2**：设计了Kill Switch硬性规则机制
  - 沙箱检测恶意代码直接判定
  - 三重认证失败+身份冒充直接判定
  - VirusTotal高检测率直接判定
- **创新点3**：实现了检测与溯源的闭环系统
  - IP地理位置追踪
  - 多DNSBL并行检测
  - 攻击链可视化展示

### 4.3 检测引擎实现
- 4.3.1 LightGBM模型训练
  - 数据集：Enron + SpamAssassin + 自建钓鱼邮件集
  - 特征标准化处理
  - 模型参数调优
  - 模型文件大小：<1MB
- 4.3.2 Kill Switch硬性规则
  ```python
  # 规则1: 沙箱检测到恶意代码
  if sandbox_detected_malware:
      return "PHISHING", 1.0, "Sandbox Detected Malware"
  
  # 规则2: 三重认证失败 + 身份冒充
  if spf_fail and dkim_fail and dmarc_fail and display_name_mismatch:
      return "PHISHING", 0.95, "Authentication Failed + Impersonation"
  
  # 规则3: VirusTotal高检测率
  if vt_detection_ratio > 0.5:
      return "PHISHING", 0.90, "Known Malicious URL"
  ```
- 4.3.3 三维度融合算法
  ```python
  # 模型预测 (权重60%)
  model_prob = model.predict(features)
  
  # URL风险 (权重40%)
  url_risk = analyze_urls(urls)
  
  # 融合计算
  confidence = model_prob * 0.6 + url_risk * 0.4
  
  # 白名单降权
  if all_urls_whitelisted:
      confidence -= 0.5
  
  # 邮件认证降权
  if spf_pass and dkim_pass and dmarc_pass:
      confidence -= 0.3
  ```
- 4.3.4 阈值判定
  - 置信度 ≥ 0.70：PHISHING（钓鱼）
  - 置信度 ≥ 0.40：SUSPICIOUS（可疑）
  - 置信度 < 0.40：SAFE（安全）

## 第六章 总结与展望

### 6.1 论文工作总结
- 实现了轻量化钓鱼邮件检测系统，适合中小企业部署
- 提出了三维度融合检测架构，准确率达到98.5%
- 实现了完整的溯源追踪功能，增强取证能力
- 系统资源占用低，无需专业运维

### 6.2 系统不足之处
- 依赖第三方API（VirusTotal）的稳定性
- 规则库需要定期更新维护
- 图片类钓鱼邮件检测能力有限
- 暂不支持分布式部署

### 6.3 未来展望
- 引入多模态模型检测图片钓鱼
- 增加自适应学习机制，减少人工干预
- 支持更多邮件协议（POP3、Exchange）
- 构建云端威胁情报共享平台
