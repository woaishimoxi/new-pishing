# 钓鱼邮件检测与溯源系统 - 项目文档

## 项目概述

本系统是一个面向中小型企业的轻量化钓鱼邮件检测与溯源系统，提供邮件安全检测、威胁情报分析、溯源追踪等功能。

## 三、服务模块详解

### 3.1 检测引擎服务 (detector.py)

**功能**：钓鱼邮件检测核心引擎

| 方法 | 功能 |
|------|------|
| `analyze()` | 主入口，分析邮件返回检测结果 |
| `_load_model()` | 加载LightGBM模型 |
| `_extract_new_features()` | 提取新模型特征 |
| `_check_kill_switch()` | 硬规则检查 |
| `_demo_predict()` | 规则评分引擎（无模型时使用） |
| `_determine_label()` | 确定最终标签 |

**检测流程**：
1. 硬规则检查（kill switch）
2. 模型预测
3. URL风险融合
4. 白名单降权
5. 邮件认证降权
6. 输出标签和置信度

### 3.2 IOC查询服务 (traceback.py)

**功能**：威胁情报查询（先本地后云端）

| 方法 | 功能 |
|------|------|
| `query_ip(ip)` | 查询IP威胁情报 |
| `query_domain(domain)` | 查询域名威胁情报 |
| `query_url(url)` | 查询URL威胁情报 |
| `_query_threatbook_ip()` | 微步IP查询API |
| `_query_threatbook_domain()` | 微步域名查询API |
| `_query_threatbook_url()` | 微步URL查询API |
| `_update_local_ioc()` | 更新本地IOC库 |

### 3.3 溯源分析服务 (traceback.py / traceback_enhanced.py)

**核心溯源 (traceback.py)**：
| 方法 | 功能 |
|------|------|
| `generate_report()` | 生成完整溯源报告 |
| `_extract_source_ip_and_path()` | 从Received链提取源IP |
| `_get_ip_geolocation()` | IP地理位置查询（百度API） |
| `_check_ip_blacklists()` | DNSBL黑名单检查 |
| `_analyze_urls()` | URL分析 |

**增强溯源 (traceback_enhanced.py)**：
| 维度 | 分析内容 |
|------|---------|
| 攻击目标 | 收件人分析、域名相似度 |
| IP来源与传播链 | 复用核心溯源数据 |
| 社会工程学 | 紧迫性诱导、权威冒充 |
| 攻击载体 | URL/附件/表单分析 |
| 攻击动机 | 经济利益/信息窃取 |

## 四、外部API服务

### 4.1 当前使用的API

| 服务 | 用途 | 响应时间 | 配置项 |
|------|------|---------|--------|
| 百度IP查询 | IP地理位置 | 139ms | 免费，无需配置 |
| 微步在线 | URL/文件/IP威胁检测 | 145-219ms | `threatbook.api_key` |
| 智谱AI | 邮件深度分析 | 国内可达 | `ai.provider=zhipu` |

### 4.2 API配置示例

```json
// config/api_config.json
{
    "threatbook": {
        "api_key": "你的微步在线API密钥",
        "api_url": "https://api.threatbook.cn/v3",
        "sandbox_enabled": true,
        "ioc_enabled": true
    },
    "ipapi": {
        "api_url": "https://opendata.baidu.com/api.php?query={ip}&co=&resource_id=6006&oe=utf8"
    },
    "ai": {
        "provider": "zhipu",
        "api_key": "你的智谱AI API密钥",
        "model": "glm-4-flash"
    }
}
```
