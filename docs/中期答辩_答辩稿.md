### 二、随机森林（RF）原理

```
小白讲解：
随机森林就像"民主投票"：
- 有100棵决策树
- 每棵树随机看部分特征、随机看部分样本
- 最终投票决定结果是"正常"还是"钓鱼"

优点：
- 不容易过拟合
- 对噪声鲁棒
- 可以评估特征重要性
```

### 三、XGBoost原理

```
小白讲解：
XGBoost就像"团队学习"：
- 第1棵树学习基本规律
- 第2棵树专门学习第1棵树的错误
- 第3棵树专门学习前两棵的错误
- ...不断改进

优点：
- 精度高
- 可以处理缺失值
- 自动学习特征组合
```

### 四、IsolationForest原理

```
小白讲解：
IsolationForest是"找异类"的专家：
- 正常邮件特征相似，聚在一起
- 钓鱼邮件特征怪异，"孤零零"分散在外

原理：
- 随机切分数据空间
- 异常点更容易被"孤立"（fewer cuts needed）
- 切分次数少的点更可能是异常
```

### 五、集成评分

```python
def ensemble_score(features_35, features_26):
    """
    融合三个模型的评分
    
    权重设计：
    - RF: 1.5 (有监督，精度高)
    - XGB: 1.5 (有监督，精度高)
    - IForest: 1.0 (无监督，发现异常)
    
    为什么RF和XGB权重更高？
    - 它们使用有标签数据训练，精度更有保障
    - IForest是无监督，作为辅助判断
    """
    scores = {
        'rf': score_with_rf(features_35),
        'xgb': score_with_xgb(features_35),
        'anomaly': score_with_anomaly_detector(features_26)
    }
    weights = {'rf': 1.5, 'xgb': 1.5, 'anomaly': 1.0}
    
    final = sum(s * weights[k] for k, s in scores.items()) / sum(weights.values())
    return final
```

## 第五部分：Kill Switch机制（建议时间：3分钟）

### 一、设计理念

```
传统检测：所有特征加权评分 → 可能漏过高危样本

Kill Switch：发现炸弹级特征 → 直接拦截

设计理念：
"宁可误杀，不可放过"
```

### 二、七大硬规则

| 序号 | 规则 | 触发条件 | 风险等级 |
|------|------|----------|----------|
| 1 | 沙箱检测到恶意代码 | 微步API返回malicious | 致命 |
| 2 | 包含可执行文件 | .exe/.scr/.bat等 | 致命 |
| 3 | 源IP在黑名单 | DNSBL标记 | 致命 |
| 4 | 认证全失败+冒充 | SPF+DKIM+DMARC全fail | 高 |
| 5 | 多个高风险URL | ≥2个高风险URL | 高 |
| 6 | 双重扩展名 | .pdf.exe等 | 高 |
| 7 | 隐藏链接+表单 | 典型钓鱼特征组合 | 中 |

### 三、代码实现

```python
def _check_kill_switch(self, features, url_analysis):
    """
    Kill Switch检查
    一旦触发，直接返回PHISHING判定
    不参与后续融合评分
    """
    # 规则1：沙箱检测到恶意代码
    if features.get('sandbox_detected'):
        return "沙箱检测到恶意代码"
    
    # 规则2：包含可执行文件
    if features.get('has_executable_attachment'):
        return "包含可执行文件附件"
    
    # 规则3：源IP在黑名单
    if features.get('first_external_ip_is_blacklisted'):
        return "源IP被列入黑名单"
    
    # ... 其他规则
    
    return None  # 未触发任何规则
```

### 四、与规则引擎的区别

| 特性 | Kill Switch | 规则引擎 |
|------|-------------|----------|
| 判定方式 | 一票否决 | 累加评分 |
| 结果 | 直接返回钓鱼 | 进入融合评分 |
| 阈值 | 无 | ≤1.0 |
| 适用场景 | 高危明确特征 | 中低风险特征 |

### 三、代码实现：IP提取

```python
def _extract_source_ip_and_path(self, received_chain):
    """
    从Received头中提取源IP
    
    邮件传播过程：
    攻击者 → 代理1 → 代理2 → 收件人
    
    每个服务器添加一个Received头
    我们要找第一个公网IP
    """
    path = []
    source_ip = "Unknown"
    
    # 从下往上遍历（从真实发送者开始）
    for line in reversed(received_chain):
        ip_match = re.search(r'\[([^\]]+)\]', line)
        if ip_match:
            ip = ip_match.group(1)
            # 跳过私有IP
            if not self._is_private_ip(ip):
                path.append(ip)
                if source_ip == "Unknown":
                    source_ip = ip
    
    return {"source_ip": source_ip, "hops": path}
```

## 答辩预演：可能被问到的问题

### Q1：为什么选择这三个模型？

**回答**：RF和XGB是互补的分类算法，RF擅长处理高维特征、XGB对异常值更鲁棒。IsolationForest是异常检测器，专门发现"不像正常邮件"的样本。三者融合可以互补长短。

### Q2：Kill Switch的权衡？

**回答**：设计哲学是"宁可误杀，不可放过"。7条规则都是基于实际攻击案例总结的，正常邮件几乎不会触发。

### Q3：如何检测未知钓鱼邮件？
