# 面向中小型企业的轻量化钓鱼邮件检测与溯源系统设计与实现

---

## 摘要

随着互联网技术的快速发展，电子邮件已成为企业日常办公的核心通信工具。然而，钓鱼邮件攻击日益猖獗，给企业信息安全带来严重威胁。据统计，全球每年因钓鱼攻击造成的经济损失高达数百亿美元，而中小企业由于缺乏专业安全团队和充足预算，往往成为攻击的主要目标。

针对上述问题，本文设计并实现了一种面向中小型企业的轻量化钓鱼邮件检测与溯源系统。该系统采用**多维度融合检测策略**，集成**轻量级机器学习模型**（RandomForest和XGBoost分类器、IsolationForest异常检测器）、**规则引擎**、**AI语义分析**和**威胁情报**，实现了对钓鱼邮件的高效检测与溯源分析。

系统的核心创新点包括：（1）设计了基于35维语义特征和26维统计特征的双维度特征提取框架；（2）提出了Kill Switch硬规则一票否决机制，针对黑名单IP、恶意附件等高危特征实现即时阻断；（3）集成大语言模型进行邮件内容语义分析，识别社会工程学攻击；（4）实现了完整的邮件溯源分析功能，包括IP追踪、DNSBL查询和攻击链还原。

实验结果表明，系统在测试集上取得了良好的检测效果，单封邮件检测时间控制在5秒以内，能够有效识别各类钓鱼邮件攻击，满足中小企业的实际应用需求。

**关键词**：钓鱼邮件检测；机器学习；轻量模型；威胁情报；溯源分析；多维度融合

## Abstract

With the rapid development of Internet technology, email has become the core communication tool for enterprise daily operations. However, phishing email attacks are becoming increasingly rampant, posing serious threats to enterprise information security. According to statistics, the global economic losses caused by phishing attacks amount to hundreds of billions of dollars annually, while small and medium-sized enterprises (SMEs) often become the main targets due to the lack of professional security teams and sufficient budgets.

To address these issues, this paper designs and implements a lightweight phishing email detection and traceback system for SMEs. The system adopts a **multi-dimensional fusion detection strategy**, integrating **lightweight machine learning models** (RandomForest and XGBoost classifiers, IsolationForest anomaly detector), **rule engine**, **AI semantic analysis**, and **threat intelligence** to achieve efficient detection and traceback analysis of phishing emails.

The core innovations of the system include: (1) A dual-dimensional feature extraction framework based on 35-dimensional semantic features and 26-dimensional statistical features; (2) A Kill Switch hard rule one-vote veto mechanism for immediate blocking of high-risk features such as blacklisted IPs and malicious attachments; (3) Integration of large language models for email content semantic analysis to identify social engineering attacks; (4) Implementation of complete email traceback analysis functions including IP tracking, DNSBL query, and attack chain reconstruction.

Experimental results show that the system achieves good detection performance on the test set, with single email detection time controlled within 5 seconds, and can effectively identify various phishing email attacks, meeting the practical application needs of SMEs.

**Keywords**: Phishing Email Detection; Machine Learning; Lightweight Model; Threat Intelligence; Traceback Analysis; Multi-dimensional Fusion

## 第1章 绪论

### 1.1 研究背景与意义

随着互联网技术的快速发展和数字化转型的深入推进，电子邮件已成为企业日常办公不可或缺的核心通信工具。据统计，全球每天发送的电子邮件数量超过3000亿封，电子邮件在企业商务沟通、内部协作、客户服务等方面发挥着重要作用。

然而，电子邮件的普及也带来了严重的安全威胁。钓鱼邮件（Phishing Email）作为一种常见的网络攻击手段，通过伪装成可信来源（如银行、政府机构、合作伙伴等），诱导用户点击恶意链接、泄露敏感信息或执行恶意操作，已成为企业面临的主要网络安全威胁之一。

根据Verizon《2024年数据泄露调查报告》，**超过90%的网络攻击始于钓鱼邮件**。IBM《2024年数据泄露成本报告》显示，全球数据泄露的平均成本达到488万美元，而其中钓鱼攻击是最常见的攻击向量之一。对于中小企业而言，由于缺乏专业的安全团队和充足的安全预算，一旦遭受钓鱼攻击，往往面临更大的经济损失和业务中断风险。

因此，开发一种**轻量化、易部署、高效率**的钓鱼邮件检测系统，对于提升中小企业的网络安全防护能力具有重要的现实意义。

### 1.2 国内外研究现状

目前，国内外学者在钓鱼邮件检测领域已开展了大量研究工作，主要的检测方法可分为以下几类：

**（1）基于规则的检测方法**

基于规则的检测方法通过预定义的规则库识别钓鱼邮件，常见的规则包括：关键词匹配、URL特征分析、发件人域名检查、邮件认证验证等。这种方法实现简单、响应速度快，但存在规则维护成本高、难以应对新型攻击的局限性。

**（2）基于机器学习的检测方法**

基于机器学习的检测方法通过训练分类模型识别钓鱼邮件，常用的算法包括：朴素贝叶斯（Naive Bayes）、支持向量机（SVM）、随机森林（Random Forest）、梯度提升树（XGBoost）等。这类方法能够自动学习特征，具有较好的泛化能力，但需要大量标注数据进行训练。

**（3）基于深度学习的检测方法**

基于深度学习的检测方法使用卷积神经网络（CNN）、循环神经网络（RNN）、Transformer等模型进行特征学习和分类。这类方法能够自动提取深层特征，在大规模数据集上表现优异，但计算资源需求大，不适合资源受限的中小企业。

**（4）基于威胁情报的检测方法**

基于威胁情报的检测方法利用外部威胁情报平台（如VirusTotal、微步在线、AlienVault等）查询URL、IP、域名、文件的威胁信息。这种方法能够快速识别已知的恶意资源，但依赖外部API，存在查询延迟和权限限制。

**现有研究的不足**：

- 深度学习方法计算资源需求大，不适合中小企业部署
- 单一检测方法的准确率有限，难以应对复杂的钓鱼攻击
- 大多数研究缺乏有效的溯源分析功能
- 缺乏针对中文钓鱼邮件的专项优化

## 第2章 相关技术介绍

### 2.1 机器学习技术

#### 2.1.1 Random Forest（随机森林）

随机森林（Random Forest）是由Leo Breiman于2001年提出的一种集成学习方法。其核心思想是通过构建多棵决策树，并将它们的预测结果进行投票或平均，从而提高模型的准确性和稳定性。

随机森林的构建过程如下：

1. 从原始训练集中使用Bootstrap抽样方法随机抽取n个样本
2. 从所有d个特征中随机选择m个特征（m << d）
3. 使用选出的样本和特征构建一棵决策树
4. 重复上述过程，构建T棵决策树
5. 对于分类问题，采用多数投票法确定最终类别

随机森林的优点包括：
- 抗过拟合能力强：由于引入了样本和特征的随机性，模型具有较好的泛化能力
- 能够处理高维特征：可以有效处理具有大量特征的数据集
- 训练速度快：各决策树之间相互独立，支持并行训练
- 对缺失值和异常值具有鲁棒性

#### 2.1.2 XGBoost（极端梯度提升）

XGBoost（eXtreme Gradient Boosting）是陈天奇于2016年提出的一种高效的梯度提升算法。它在传统梯度提升决策树（GBDT）的基础上进行了多项优化，包括正则化、并行计算、缺失值处理等。

XGBoost的目标函数定义为：

$$Obj = \sum_{i=1}^{n} L(y_i, \hat{y}_i) + \sum_{k=1}^{K} \Omega(f_k)$$

其中，$L(y_i, \hat{y}_i)$为损失函数，$\Omega(f_k)$为正则化项，用于控制模型复杂度。

XGBoost的优点包括：
- 精度高：通过正则化有效防止过拟合
- 速度快：支持特征级并行计算
- 灵活性强：支持自定义损失函数和评估指标
- 内置缺失值处理：自动学习缺失值的最优分裂方向

#### 2.1.3 Isolation Forest（孤立森林）

Isolation Forest（孤立森林）是刘飞等人于2008年提出的一种无监督异常检测算法。其核心思想是：异常样本由于与大多数样本不同，在随机分割时更容易被孤立。

Isolation Forest的算法流程如下：

1. 随机选择一个特征
2. 在该特征的最大值和最小值之间随机选择一个分割点
3. 根据分割点对数据进行划分
4. 递归执行上述过程，直到样本被完全隔离或达到最大深度
5. 异常样本的路径长度通常较短

Isolation Forest的优点包括：
- 计算复杂度低：时间复杂度为O(nlogn)
- 适合高维数据：不需要计算距离或密度
- 不需要标注数据：属于无监督学习方法
- 对异常值敏感：能够有效检测离群点

### 2.2 自然语言处理技术

### 2.4 邮件协议与认证机制

#### 2.4.1 SPF（Sender Policy Framework）

SPF是一种邮件认证机制，用于验证发送邮件的服务器IP是否被域名所有者授权。SPF通过在DNS中发布TXT记录，指定哪些IP地址可以代表该域名发送邮件。

SPF验证结果包括：
- **Pass**：发送IP被授权
- **Fail**：发送IP未被授权
- **SoftFail**：发送IP可能未被授权
- **Neutral**：无法确定

#### 2.4.2 DKIM（DomainKeys Identified Mail）

DKIM是一种邮件认证机制，通过在邮件头中添加数字签名，验证邮件内容在传输过程中是否被篡改。

DKIM的工作流程：
1. 发送方使用私钥对邮件进行签名
2. 签名信息添加到邮件头的DKIM-Signature字段
3. 接收方从DNS获取发送方的公钥
4. 使用公钥验证签名的有效性

#### 2.4.3 DMARC（Domain-based Message Authentication, Reporting and Conformance）

DMARC是一种综合邮件认证策略，结合SPF和DKIM的结果，为域名所有者提供邮件认证策略管理和报告功能。

DMARC策略包括：
- **none**：仅监控，不做处理
- **quarantine**：将未通过认证的邮件放入垃圾箱
- **reject**：拒绝未通过认证的邮件

### 4.3 特征工程设计

#### 4.3.1 35维语义特征

35维语义特征主要用于RF分类器和XGB分类器，涵盖以下类别：

| 类别 | 特征数 | 具体特征 |
|------|--------|----------|
| 主题特征 | 6 | urgency_level（紧急程度）、threatening（威胁性）、seductive（诱惑性）、emergency（紧急行动）、sentiment_score（情感分数）、sentiment_label（情感标签） |
| 发件人特征 | 2 | impersonation_type（冒充类型）、email_anomaly（邮箱异常） |
| 正文特征 | 16 | word_count（词数）、url_count（URL数）、spelling_errors（拼写错误）、grammar_errors（语法错误）、suspicious_keywords（可疑关键词）、urgency_words（紧急词）、personal_info_request（个人信息请求）、financial_request（财务请求）、text_complexity（文本复杂度）、similarity（相似度）、language（语言）、obfuscated（混淆）、otp_request（验证码请求）、phishing_cta（钓鱼CTA）、text_sentiment（情感）、text_sentiment_score（情感分数） |
| URL特征 | 11 | domain_length（域名长度）、dot_count（点号数）、contains_ip（是否IP地址）、contains_at（是否包含@）、contains_hyphen（是否包含连字符）、path_length（路径长度）、subdomains_count（子域名数）、tld（顶级域名）、query_params_count（参数数）、has_suspicious_params（可疑参数）、suspicious_params_count（可疑参数数） |

#### 4.3.2 26维统计特征

26维统计特征主要用于IsolationForest异常检测器，涵盖以下类别：

| 类别 | 特征数 | 具体特征 |
|------|--------|----------|
| 基础统计 | 6 | num_chars（字符数）、num_lines（行数）、num_urls（URL数）、subject_len（主题长度）、keyword_hit_count（关键词命中）、brand_hit_count（品牌命中） |
| URL特征 | 4 | high_risk_url_count（高风险URL）、anchor_mismatch_count（链接不匹配）、unique_domains（唯一域名）、has_ip_url（是否有IP地址URL） |
| HTML特征 | 3 | has_html（是否有HTML）、has_script_or_form（是否有脚本/表单）、has_attachment_hint（附件提示） |
| 钓鱼模式 | 8 | fake_sender_score（伪造发件人）、has_base64（是否有base64）、base64_blocks（base64块数）、chinese_keyword_count（中文关键词）、attachment_risk_score（附件风险）、received_count（Received头数）、boundary_count（boundary数）、has_x_mailer（是否有X-Mailer） |
| 邮件认证 | 5 | spf_fail（SPF失败）、dkim_fail（DKIM失败）、dmarc_fail（DMARC失败）、dkim_present（是否有DKIM）、spf_present（是否有SPF） |

### 4.4 融合检测设计

系统采用加权融合策略，各检测模块权重如下：

| 模块 | 权重 | 说明 |
|------|------|------|
| RF分类器 | 1.5 | 基于35维语义特征的随机森林模型 |
| XGB分类器 | 1.5 | 基于35维语义特征的梯度提升模型 |
| 异常检测器 | 1.0 | 基于26维统计特征的孤立森林模型 |
| 规则引擎 | 1.0 | 基于关键词和模式匹配的规则检测 |
| AI分析 | 1.2 | 大语言模型语义分析 |
| URL分析 | 1.0 | URL风险特征分析 |

融合公式：

$$最终得分 = \frac{\sum_{i=1}^{n}(模块得分_i \times 权重_i)}{\sum_{i=1}^{n}权重_i}$$

其中，n为有效模块数量（部分模块可能未激活）。

### 4.5 Kill Switch设计

Kill Switch是一种一票否决机制，当检测到以下高危特征时，直接判定为PHISHING，不参与融合评分：

| 序号 | 高危特征 | 说明 |
|------|----------|------|
| 1 | 沙箱检测到恶意代码 | 附件沙箱分析发现恶意行为 |
| 2 | 包含可执行文件附件 | 附件为.exe、.bat等可执行文件 |
| 3 | 源IP在黑名单中 | 源IP被DNSBL黑名单标记 |
| 4 | 邮件认证全部失败+发件人冒充 | SPF/DKIM/DMARC均失败且发件人冒充 |
| 5 | 多个高风险URL | 检测到2个及以上高风险URL |
| 6 | 双重扩展名附件 | 附件使用双重扩展名（如invoice.pdf.exe） |
| 7 | 隐藏链接+表单组合 | 同时包含隐藏链接和表单 |

Kill Switch的设计目的是确保关键威胁能够被即时阻断，避免因融合评分的稀释效应导致漏检。

### 4.6 阈值设计

系统采用双阈值设计，将检测结果分为三个等级：

| 等级 | 阈值 | 说明 |
|------|------|------|
| PHISHING | >= 0.60 | 判定为钓鱼邮件 |
| SUSPICIOUS | >= 0.35 | 判定为可疑邮件，建议人工复核 |
| SAFE | < 0.35 | 判定为安全邮件 |

阈值的设置遵循"宁可误报，不可漏报"的原则，以确保钓鱼邮件能够被有效识别。

```python
class EmailParserService:
    """邮件解析服务"""
    
    def parse(self, raw_email: str) -> Dict[str, Any]:
        """解析原始邮件，提取结构化数据"""
        msg = email.message_from_string(raw_email)
        
        # 提取邮件头信息
        from_display_name, from_email = self._parse_email_address(msg.get("From"))
        subject = self._decode_mime_header(msg.get("Subject", ""))
        to_addr = self._decode_mime_header(msg.get("To", ""))
        
        # 提取邮件正文和附件
        body, html_body, attachments = self._extract_body_and_attachments(msg)
        
        # 提取URL
        urls = self._extract_urls(body)
        if html_body:
            urls.extend(self._extract_urls_from_html(html_body))
        
        # 提取邮件头认证结果
        headers = self._extract_headers(msg)
        
        return {
            "from_display_name": from_display_name,
            "from_email": from_email,
            "subject": subject,
            "to": to_addr,
            "body": body,
            "html_body": html_body,
            "urls": urls,
            "attachments": attachments,
            "headers": headers
        }
```

#### 5.2.2 检测引擎模块

检测引擎模块是系统的核心，负责协调各检测模块并进行融合评分：

```python
class DetectionService:
    """检测服务"""
    
    def analyze(
        self,
        email_data: Dict,
        features: Dict,
        ai_analysis: Dict = None,
        url_analysis: Dict = None
    ) -> Tuple[str, float, str, Dict]:
        """多维度融合检测"""
        
        # 初始化模型得分
        model_scores = {
            'rf': None, 'xgb': None, 'anomaly': None,
            'rule': 0.0, 'ai': None, 'url': None
        }
        
        # 1. Kill Switch检查
        kill_reason = self._check_kill_switch(features, url_analysis)
        if kill_reason:
            return "PHISHING", 0.99, kill_reason, model_scores
        
        # 2. 轻量模型检测
        features_35d = extract_features(email_data, '35d')
        features_26d = extract_features(email_data, '26d')
        
        model_scores['rf'] = score_with_rf(features_35d)
        model_scores['xgb'] = score_with_xgb(features_35d)
        model_scores['anomaly'] = score_with_anomaly_detector(features_26d)
        
        # 3. 规则引擎评分
        model_scores['rule'] = self._rule_engine_score(features, email_data)
        
        # 4. AI分析评分
        if ai_analysis:
            model_scores['ai'] = ai_analysis.get('risk_score', 0) / 100.0
        
        # 5. URL分析评分
        if url_analysis:
            model_scores['url'] = url_analysis.get('max_risk_score', 0) / 100.0
        
        # 6. 加权融合
        weights = {'rf': 1.5, 'xgb': 1.5, 'anomaly': 1.0, 'rule': 1.0, 'ai': 1.2, 'url': 1.0}
        final_score = self._fusion_scores(model_scores, weights)
        
        # 7. 阈值判断
        if final_score >= 0.60:
            return "PHISHING", final_score, self._build_reason(model_scores), model_scores
        elif final_score >= 0.35:
            return "SUSPICIOUS", final_score, self._build_reason(model_scores), model_scores
        else:
            return "SAFE", final_score, "未检测到明显钓鱼特征", model_scores
```

### 7.1 工作总结

本文设计并实现了一种面向中小型企业的轻量化钓鱼邮件检测与溯源系统。主要完成了以下工作：

1. **设计了多维度融合检测框架**：集成轻量模型（RF、XGB、IsolationForest）、规则引擎、AI语义分析和威胁情报，实现了6个维度的融合检测

2. **实现了双维度特征提取**：设计了35维语义特征和26维统计特征的特征提取框架，能够从多个角度刻画邮件的风险特征

3. **设计了Kill Switch硬规则**：针对黑名单IP、恶意附件等高危特征设计了一票否决机制，确保关键威胁即时阻断

4. **集成了AI语义分析**：利用大语言模型（通义千问）分析邮件内容，能够识别传统方法难以检测的社会工程学攻击

5. **实现了完整的溯源分析**：包括IP地理位置查询、DNSBL黑名单查询、WHOIS域名查询和攻击链还原

6. **开发了完整的Web界面**：包括大屏展示、检测面板、报告详情、系统配置等页面

### 7.2 创新点

本文的主要创新点包括：

1. **轻量化设计**：采用轻量级机器学习模型（RandomForest、XGBoost、IsolationForest），降低计算资源需求，适合中小企业部署

2. **多维度融合**：融合6个检测维度，通过加权融合策略提高检测准确率，实验证明融合检测比单一检测方法提升约16个百分点

3. **AI语义分析**：集成大语言模型进行邮件内容深度分析，能够识别传统关键词匹配难以检测的社会工程学攻击

4. **Kill Switch机制**：针对高危特征设计一票否决机制，确保关键威胁即时阻断，避免漏检

5. **完整溯源分析**：实现从邮件源IP到攻击链的完整溯源分析，支持并行DNSBL查询

### 7.3 不足与展望

本系统仍存在以下不足：

1. **特征工程可优化**：当前特征提取依赖外部API（WHOIS），可能导致查询延迟，后续可考虑使用本地缓存或预计算

2. **模型训练数据有限**：当前使用的模型基于公开数据集训练，后续可收集更多真实钓鱼邮件数据进行再训练

3. **缺乏用户认证**：系统当前无用户认证机制，后续可添加JWT认证和权限管理

4. **单机部署限制**：系统当前为单机部署，后续可支持分布式部署以提高并发能力

未来工作方向：

1. **引入联邦学习**：实现多企业协同检测，在保护数据隐私的前提下共享威胁情报

2. **集成图像识别**：检测邮件中的图片钓鱼（如二维码钓鱼、截图钓鱼）

3. **支持更多邮件格式**：除.eml外，支持.msg、.mbox等格式

4. **实时流处理**：引入Apache Kafka等流处理框架，支持实时邮件检测

5. **移动端支持**：开发移动端应用，支持随时随地查看检测结果
