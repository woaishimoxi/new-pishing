## 系统概述

本系统是一个面向中小型企业的轻量化钓鱼邮件检测与溯源系统，旨在帮助企业快速识别钓鱼邮件、追踪攻击来源，并提供详细的分析报告。系统采用多维度特征分析、机器学习模型检测、规则引擎评分相结合的方式，实现对钓鱼邮件的高效检测。

### 核心特性

- **多渠道邮件获取**：支持手动输入、文件上传、IMAP自动收取三种方式
- **多维度特征提取**：从邮件头、URL、文本、附件、HTML等维度提取特征
- **智能检测引擎**：融合机器学习模型与规则引擎的双重检测
- **深度溯源分析**：五维度溯源分析，追踪攻击来源与动机
- **云沙箱集成**：支持VirusTotal等第三方沙箱进行附件检测
- **AI辅助分析**：集成多种AI服务，提供深度分析能力

## 完整处理流程

### 一、邮件获取阶段

系统支持三种邮件获取方式：

#### 1.1 手动输入

用户直接在检测页面粘贴原始邮件源码（包含完整邮件头和正文）。

```
POST /api/detection/analyze
Content-Type: application/json

{
    "email": "原始邮件源码...",
    "source": "手动输入"
}
```

#### 1.2 文件上传

支持上传 `.eml` 或 `.msg` 格式的邮件文件。

```
POST /api/detection/upload
Content-Type: multipart/form-data

file: email.eml
```

#### 1.3 IMAP自动收取

通过IMAP协议从第三方邮件服务器自动收取邮件。

```
POST /api/email/fetch
Content-Type: application/json

{
    "server": "imap.example.com",
    "port": 993,
    "username": "user@example.com",
    "password": "password",
    "folder": "INBOX"
}
```

### 二、邮件解析阶段

通过 `EmailParserService` 将原始邮件按维度拆分：

#### 2.1 解析流程

```
原始邮件 → EmailParserService.parse() → 结构化数据
```

#### 2.2 解析维度

| 维度 | 提取内容 |
|------|---------|
| 邮件头 | 发件人、收件人、抄送、主题、日期、Message-ID |
| 认证信息 | SPF、DKIM、DMARC验证结果 |
| 路由信息 | Received链、X-Mailer、Return-Path |
| 文本内容 | 纯文本正文、HTML正文 |
| URL链接 | 所有URL、HTML中的链接、表单action |
| 附件 | 文件名、类型、大小、MD5/SHA256哈希 |
| HTML特征 | 隐藏链接、表单、iframe、外部脚本 |

#### 2.3 解析结果示例

```json
{
    "from": "sender@example.com",
    "from_display_name": "Example Sender",
    "from_email": "sender@example.com",
    "to": "recipient@company.com",
    "subject": "Important Notice",
    "body": "纯文本正文内容...",
    "html_body": "<html>...</html>",
    "urls": ["https://example.com/link1", "https://example.com/link2"],
    "attachments": [
        {
            "filename": "document.pdf",
            "content_type": "application/pdf",
            "size": 102400,
            "md5": "abc123..."
        }
    ],
    "headers": {
        "spf_result": "pass",
        "dkim_result": "pass",
        "dmarc_result": "pass"
    },
    "html_links": [...],
    "html_forms": [...]
}
```

### 四、检测分析阶段

#### 4.1 模型检测

`DetectionService` 使用预训练的轻量化模型进行检测：

```
特征向量 → LightGBM模型 → 模型置信度
```

**支持的模型格式**：
- `.pkl` - Pickle序列化模型
- `.txt` - LightGBM文本格式模型

**模型特征列**：
```python
NEW_FEATURE_COLUMNS = [
    'text_length', 'word_count', 'uppercase_ratio', 'exclamation_count',
    'question_count', 'digit_ratio', 'special_char_ratio', 'url_count',
    'ip_url_count', 'shortened_url_count', 'urgent_word_count',
    'financial_word_count', 'phishing_pattern_count', 'email_count',
    'html_tag_count', 'link_count', 'form_count'
]
```

#### 4.2 URL风险分析

`URLAnalyzerService` 对邮件中的URL进行深度分析：

**分析维度**：
- 域名信誉检查
- 白名单匹配
- 短链接检测
- IP地址URL检测
- 可疑参数检测

**风险等级**：
| 等级 | 说明 | 评分范围 |
|------|------|---------|
| HIGH | 高风险 | ≥80 |
| MEDIUM | 中风险 | 40-79 |
| LOW | 低风险 | <40 |

#### 4.3 云沙箱分析

`SandboxAnalyzerService` 调用第三方沙箱检测附件：

**触发条件**：
- 文件类型为可执行文件或文档
- 文件大小在限制范围内
- 已配置VirusTotal API Key

**分析流程**：
```
附件 → VirusTotal API → 检测报告
```

**返回结果**：
```json
{
    "sandbox_detected": true,
    "sandbox_report": {
        "positives": 15,
        "total": 70,
        "detection_ratio": 0.21
    }
}
```

### 五、评分融合阶段

#### 5.1 融合公式

```
最终置信度 = 模型评分 × 0.6 + URL风险评分 × 0.4 + 规则评分调整
```

#### 5.2 硬规则触发（Kill Switch）

以下情况直接判定为钓鱼邮件（置信度0.99）：

| 规则 | 说明 |
|------|------|
| Sandbox Detected Malware | 沙箱检测到恶意代码 |
| Authentication Failed + Impersonation | SPF/DKIM/DMARC全部失败 + 显示名欺骗 |
| Known Malicious URL | VirusTotal检测率 > 50% |
| High Risk URL Detected | 高风险URL（评分≥80） |

#### 5.3 置信度阈值

| 置信度范围 | 判定结果 | 说明 |
|-----------|---------|------|
| ≥ 0.7 | PHISHING | 钓鱼邮件 |
| 0.4 - 0.7 | SUSPICIOUS | 可疑邮件 |
| < 0.4 | SAFE | 正常邮件 |

#### 5.4 白名单保护机制

- 所有URL在白名单中 → 置信度降低0.5
- 大部分URL在白名单中 → 置信度降低0.3
- 邮件认证全部通过 → 置信度降低0.3
- 发件人域名与链接域名一致 → 置信度降低0.2

### 六、溯源分析阶段

`TracebackAnalyzer` 从5个维度进行深度溯源：

#### 6.1 维度一：攻击目标

分析收件人信息，判断攻击类型：

| 收件人数量 | 攻击类型 | 风险等级 |
|-----------|---------|---------|
| 1 | 定向攻击（针对性钓鱼） | medium |
| 2-5 | 小范围攻击（部门级钓鱼） | medium |
| >5 | 广撒网攻击（批量钓鱼） | low |

**关键岗位检测**：
检测收件人邮箱是否包含 `admin`、`finance`、`hr`、`ceo` 等关键词。

#### 6.2 维度二：IP来源与传播链

**分析内容**：
- 提取邮件头中的Received链
- 识别源IP地址
- 查询IP地理位置
- 构建传播路径

**输出示例**：
```json
{
    "source_ip": "192.168.1.1",
    "geolocation": {
        "country": "United States",
        "city": "Los Angeles",
        "isp": "Example ISP"
    },
    "hops": [
        {"step": 1, "ip": "192.168.1.1", "server": "mail.example.com"},
        {"step": 2, "ip": "10.0.0.1", "server": "relay.company.com"}
    ],
    "full_path": "mail.example.com → relay.company.com"
}
```

#### 6.3 维度三：攻击特性（社会工程学）

**检测类别**：

| 类别 | 英文关键词 | 中文关键词 |
|------|-----------|-----------|
| urgency（紧迫性） | urgent, immediately, expires | 紧急, 立即, 过期 |
| fear（恐惧诱导） | suspended, blocked, hacked | 冻结, 封禁, 被盗 |
| reward（利益诱惑） | congratulations, winner, prize | 恭喜, 中奖, 奖励 |
| authority（权威伪装） | administrator, support team | 管理员, 客服 |
| generic（通用称呼） | dear user, dear customer | 尊敬的用户 |

**风险判定**：
- 检测到≥3种类别 → 高风险
- 检测到1-2种类别 → 中风险
- 未检测到 → 低风险

#### 6.4 维度四：攻击载体

**检测内容**：

| 载体类型 | 检测方法 |
|---------|---------|
| 恶意链接 | 短链接、IP地址URL、可疑域名 |
| 可疑附件 | 危险扩展名（.exe, .scr, .bat等） |
| 二维码 | 检测"qr"、"二维码"关键词 |
| 信息窃取 | 检测密码、银行卡、身份证等请求 |

#### 6.5 维度五：攻击动机

**动机类型**：

| 动机 | 说明 | 判定依据 |
|------|------|---------|
| credential_theft | 窃取凭证 | 链接指向登录页面 |
| malware_delivery | 植入恶意软件 | 包含可疑附件 |
| financial_fraud | 财务诈骗 | 利诱手段诱导用户 |
| data_exfiltration | 数据窃取 | 请求敏感信息 |

#### 6.6 风险评分计算

```
风险评分 = 攻击目标(0.1) + IP来源(0.2) + 社会工程学(0.25) + 攻击载体(0.35) + 攻击动机(0.1)
```

### 七、报告生成阶段

#### 7.1 报告内容

| 模块 | 内容 |
|------|------|
| 基础概览 | 判定结果、置信度、风险评分、主要致因、处理建议 |
| 邮件头分析 | 发件人详情、认证状态、路由路径 |
| 链接与附件分析 | URL风险详情、附件安全状态 |
| 云沙箱分析 | 沙箱检测结果（如有） |
| 技术特征分析 | 模块评分、风险指标、特征详情 |
| 用户确认区域 | 对可疑邮件提供人工确认选项 |

#### 7.2 模块评分

系统计算5个模块的独立评分：

```python
module_scores = {
    'header': 0.0,    # 邮件头特征评分
    'url': 0.0,       # URL特征评分
    'text': 0.0,      # 文本特征评分
    'attachment': 0.0, # 附件特征评分
    'html': 0.0       # HTML特征评分
}
```

#### 7.3 报告存储

检测结果保存到SQLite数据库：

```python
alert_id = db.save_alert(
    parsed,           # 解析后的邮件数据
    label,            # 判定结果
    confidence,       # 置信度
    traceback_report, # 溯源报告
    source,           # 来源
    raw_email_stored, # 原始邮件
    email_uid         # 邮件UID
)
```

### 八、AI深度分析（可选）

当邮件被标记为可疑时，用户可调用AI进行深度分析。

#### 8.1 支持的AI服务

| 服务商 | 模型 | 说明 |
|--------|------|------|
| OpenAI | GPT-4 | OpenAI官方API |
| 百度 | 文心一言 | 百度智能云 |
| 阿里 | 通义千问 | 阿里云 |
| 智谱 | ChatGLM | 智谱AI |
| 月之暗面 | Kimi | Moonshot AI |

#### 8.2 分析维度

AI从以下维度进行专业分析：

1. **发件人分析**：邮箱地址可疑性、显示名匹配度、域名信誉
2. **内容分析**：紧急语言、敏感信息请求、拼写语法错误
3. **链接分析**：URL可疑性、短链接、域名模仿
4. **邮件头分析**：认证状态、路由异常、伪造痕迹

#### 8.3 返回结果

```json
{
    "is_phishing": true,
```
