## 1. 项目概述与需求分析

### 1.1 业务背景

钓鱼邮件是网络攻击的主要载体之一，据统计，超过90%的网络攻击始于钓鱼邮件。中小型企业由于缺乏专业的安全团队和技术手段，面临着严峻的邮件安全威胁。本系统旨在为中小型企业提供一套轻量化、高精度、易部署的钓鱼邮件检测解决方案。

| 威胁情报 | 微步在线API集成 | P1 |

## 3. 数据预处理方法

### 3.1 邮件解析流程

```
原始EML文件
    │
    ├── 1. MIME解析 → 提取邮件头、正文、附件
    ├── 2. 编码处理 → UTF-8解码、处理乱码
    ├── 3. HTML净化 → 去除脚本、提取纯净文本
    ├── 4. URL提取 → 正则匹配提取链接
    └── 5. 邮件头解析 → SPF/DKIM/DMARC
```

### 3.2 邮件头解析

```python
def parse_email_headers(content: str) -> Dict:
    """解析邮件头信息"""
    headers = {}
    
    # From字段解析
    from_match = re.search(r'^From:\s*(.+)$', content, re.MULTILINE)
    if from_match:
        headers['from'] = from_match.group(1)
        # 提取邮箱域名
        email_match = re.search(r'<(.+@.+)>', from_match.group(1))
        if email_match:
            headers['from_email'] = email_match.group(1)
        headers['from_domain'] = headers.get('from_email', '').split('@')[-1]
    
    # SPF/DKIM/DMARC解析
    auth_results = re.search(
        r'Authentication-Results:.*?(spf=[a-z]+).*?(dkim=[a-z]+).*?(dmarc=[a-z]+)',
        content, re.IGNORECASE | re.DOTALL
    )
    if auth_results:
        headers['spf_result'] = auth_results.group(1).split('=')[1]
        headers['dkim_result'] = auth_results.group(2).split('=')[1]
        headers['dmarc_result'] = auth_results.group(3).split('=')[1]
    
    # Received链解析（用于溯源）
    received_headers = re.findall(r'^Received:\s*(.+)$', content, re.MULTILINE)
    headers['received_chain'] = received_headers
    
    return headers
```

### 3.3 文本预处理

```python
def clean_html_body(html_content: str) -> str:
    """HTML正文净化"""
    # 1. 去除CSS样式
    text = re.sub(r'<style[^>]*>.*?</style>', '', html_content, 
                  flags=re.DOTALL | re.IGNORECASE)
    
    # 2. 去除JavaScript
    text = re.sub(r'<script[^>]*>.*?</script>', '', text, 
                  flags=re.DOTALL | re.IGNORECASE)
    
    # 3. 去除HTML标签
    text = re.sub(r'<[^>]+>', ' ', text)
    
    # 4. 去除多余空白
    text = re.sub(r'\s+', ' ', text).strip()
    
    return text
```

### 4.2 模型1：随机森林分类器（RF）

**算法原理**：
随机森林是一种集成学习方法，通过构建多棵决策树并取其投票结果的平均值来进行分类。每棵树在训练时使用随机抽样和随机特征子集，增加了模型的多样性，提高了泛化能力。

**超参数配置**：

```python
rf_params = {
    'n_estimators': 100,        # 决策树数量
    'max_depth': 20,            # 最大深度
    'min_samples_split': 10,     # 内部节点再划分所需最小样本数
    'min_samples_leaf': 5,       # 叶子节点最少样本数
    'random_state': 42,         # 随机种子
    'n_jobs': -1,               # 并行计算
}
```

**训练代码**：

```python
from sklearn.ensemble import RandomForestClassifier
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler
```

### 4.3 模型2：XGBoost分类器

**算法原理**：
XGBoost是梯度提升决策树的高效实现，通过迭代地添加弱分类器来最小化损失函数。每棵树都试图纠正前一棵树的错误，最终通过加权求和得到最终预测。

**超参数配置**：

```python

    'colsample_bytree': 0.8,   # 列采样比例

    'n_jobs': -1,

```

**训练代码**：

```python

```

### 4.4 模型3：IsolationForest异常检测

**算法原理**：
IsolationForest是一种无监督异常检测算法，基于"异常点更容易被随机切分隔离"的假设。异常点在决策树中通常具有更短的路径长度，因此可以通过路径长度来识别异常。

**超参数配置**：


## 6. 核心检测算法

### 6.1 检测服务架构

```python
class DetectionService:
    """钓鱼邮件检测服务"""
    
    def analyze(self, email_data, features, ai_analysis=None, url_analysis=None):
        """
        多阶段检测流程：
        1. Kill Switch检查（快速拦截高危邮件）
        2. 轻量模型评分（RF/XGB/IForest）
        3. 规则引擎评分
        4. AI语义评分（可选）
        5. URL分析评分（可选）
        6. 融合评分与判定
        """
        
        # 阶段1：Kill Switch检查
        kill_switch_result = self._check_kill_switch(email_data)
        if kill_switch_result['triggered']:
            return self._format_result('PHISHING', 1.0, 
                                       kill_switch_result['reason'], 
                                       {'kill_switch': kill_switch_result})
        
        # 阶段2-5：各模块评分
        model_scores = self._score_all_modules(features, ai_analysis, url_analysis)
        
        # 阶段6：融合评分
        final_score = self._fuse_scores(model_scores)
        
        # 阶段7：阈值判定
        label, confidence = self._classify(final_score)
        
        return self._format_result(label, confidence, reason, model_scores)
```

### 6.2 Kill Switch机制

**设计原理**：
对于某些高置信度的钓鱼特征，无需经过机器学习模型即可直接判定为钓鱼邮件。这既能提高检测速度，又能避免模型误判导致的安全事故。

**触发规则**：

| 序号 | 规则 | 风险等级 | 说明 |
|------|------|----------|------|
| 1 | 沙箱检测到恶意代码 | 致命 | 直接判定 |
| 2 | 包含可执行文件附件 | 致命 | .exe/.scr/.bat等 |
| 3 | 源IP在黑名单中 | 致命 | DNSBL标记 |
| 4 | SPF+DKIM+DMARC全失败且冒充 | 高 | 认证伪造 |
| 5 | 多个高风险URL | 高 | ≥2个可疑URL |
| 6 | 双重扩展名附件 | 高 | 如.pdf.exe |
| 7 | 隐藏链接+表单组合 | 中 | 典型钓鱼特征 |

```python
def _check_kill_switch(self, email_data: Dict) -> Dict:
    """Kill Switch检查"""
    reasons = []
    
    # 1. 检查附件
    attachments = email_data.get('attachments', [])
    for att in attachments:
        filename = att.get('filename', '').lower()
        # 可执行文件
        if any(filename.endswith(ext) for ext in ['.exe', '.scr', '.bat', '.cmd', '.vbs', '.ps1']):
            reasons.append(f"包含可执行文件附件: {filename}")
        
        # 双重扩展名
        if filename.count('.') >= 2:
            last_ext = filename.split('.')[-1]
            second_last_ext = filename.split('.')[-2]
            if second_last_ext in ['pdf', 'doc', 'docx', 'xls', 'xlsx']:
                reasons.append(f"双重扩展名: {filename}")
    
    # 2. 检查黑名单IP
    source_ip = email_data.get('source_ip', '')
    if source_ip in self.blacklisted_ips:
        reasons.append(f"源IP在黑名单中: {source_ip}")
    
    # 3. 检查认证结果
    auth = email_data.get('auth_results', {})
    if auth.get('spf') == 'fail' and auth.get('dkim') == 'fail' and auth.get('dmarc') == 'fail':
        if self._check_impersonation(email_data):
            reasons.append("SPF/DKIM/DMARC全失败且存在冒充")
    
    return {
        'triggered': len(reasons) > 0,
        'reasons': reasons
    }
```

### 6.3 模型评分流程

```python
def _score_all_modules(self, features: Dict, ai_analysis: Dict, url_analysis: Dict) -> Dict:
    """各模块评分"""
    scores = {}
    
    # RF分类器评分
    if self.model_available:
        features_35 = features.get('features_35d', [])
        if len(features_35) == 35:
            scores['rf'] = score_with_rf(features_35)
            scores['xgb'] = score_with_xgb(features_35)
        
        features_26 = features.get('features_26d', [])
        if len(features_26) == 26:
            scores['anomaly'] = score_with_anomaly_detector(features_26)
    
    # 规则引擎评分
    scores['rule'] = self._score_with_rules(features)
    
    # AI评分（可选）
    if ai_analysis and ai_analysis.get('success'):
        scores['ai'] = ai_analysis.get('phishing_score', 0.5)
    
    # URL评分（可选）
    if url_analysis:
        scores['url'] = url_analysis.get('risk_score', 0.5)
    
    return scores
```

## 7. 融合策略与判定机制

### 7.1 加权融合算法

**公式**：

$$S_{final} = \frac{\sum_{i=1}^{n} w_i \cdot s_i}{\sum_{i=1}^{n} w_i}$$

其中：
- $S_{final}$：最终融合得分
- $w_i$：第i个模块的权重
- $s_i$：第i个模块的得分

**权重配置**：

| 模块 | 权重 | 权重系数 | 说明 |
|------|------|----------|------|
| RF分类器 | 1.5 | RF_XGB_WEIGHT = 1.5 | 语义特征分类 |
| XGB分类器 | 1.5 | RF_XGB_WEIGHT = 1.5 | 语义特征分类 |
| 异常检测器 | 1.0 | ANOMALY_WEIGHT = 1.0 | 统计异常检测 |
| 规则引擎 | 1.0 | RULE_WEIGHT = 1.0 | 规则匹配 |
| AI分析 | 1.2 | AI_WEIGHT = 1.2 | 语义理解 |
| URL分析 | 1.0 | URL_WEIGHT = 1.0 | URL风险评估 |

**权重设计理由**：
- RF和XGB使用相同的35维语义特征，权重相等
- AI分析的权重略低于RF/XGB，因为其结果受模型质量影响大
- 异常检测和规则引擎权重为1.0，作为基础评分
- URL分析独立于内容，用于补充

### 7.2 判定阈值

| 判定结果 | 阈值范围 | 置信度 | 建议操作 |
|----------|----------|--------|----------|
| PHISHING | ≥ 0.60 | 高置信度 | 直接拦截 |
| SUSPICIOUS | ≥ 0.35, < 0.60 | 中等置信度 | 标记+人工复核 |
| SAFE | < 0.35 | 高置信度 | 正常投递 |

```python
DNSBL_SERVERS = [
    ('zen.spamhaus.org', 'Spamhaus'),      # 最权威
    ('b.barracudacentral.org', 'Barracuda'),
    ('bl.spamcop.net', 'SpamCop'),
    ('dnsbl.sorbs.net', 'SORBS'),
    ('psbl.surriel.com', 'PSBL'),
]

def check_ip_blacklist(ip: str) -> Dict:
    """DNSBL黑名单检查"""
    results = []
    
    # 反向IP
    reversed_ip = '.'.join(ip.split('.')[::-1])
    
    for server, name in DNSBL_SERVERS:
        try:
            query_host = f"{reversed_ip}.{server}"
            answers = socket.gethostbyname_ex(query_host)
            results.append({
                'server': name,
                'listed': True,
                'response': answers[2]  # 返回原因码
            })
        except socket.gaierror:
            results.append({
                'server': name,
                'listed': False
            })
    
    return {
        'ip': ip,
        'listed': any(r['listed'] for r in results),
        'listings': [r for r in results if r['listed']],
        'total_checks': len(DNSBL_SERVERS),
        'positive_checks': sum(1 for r in results if r['listed'])
    }
```

## 11. 技术难点与解决方案

### 11.1 特征提取的可扩展性

**难点**：不同邮件格式差异大，特征提取可能遗漏关键信息。

**解决方案**：
1. 采用分层特征提取：基础特征 + 扩展特征 + 语义特征
2. 实现特征验证机制，确保特征向量维度正确
3. 对于无法提取的特征使用合理默认值

```python
def extract_features_safe(parsed_email: Dict) -> List[float]:
    """安全的特征提取"""
    features_35 = extract_phishmmf_features_35d(parsed_email)
    if len(features_35) != 35:
        features_35 = [0.0] * 35
    
    features_26 = extract_iforest_features_26d(parsed_email)
    if len(features_26) != 26:
        features_26 = [0.0] * 26
    
    return features_35, features_26
```
